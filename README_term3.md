# **SketchToSpec: 손그림으로 요구사항 명세서 및 다이어그램을 자동 생성하는 에이전트**

## **프로젝트 개요**
**SketchToSpec**은 사용자가 제공한 **손그림**을 분석하고, 이를 바탕으로 **요구사항 명세서**와 **다이어그램**을 자동으로 생성하는 **에이전트 시스템**입니다. 주제와 UI 컴포넌트를 손그림으로 입력받고, 이를 기반으로 **텍스트 형식의 요구사항 명세서**와 **ASCII 아트 다이어그램**을 생성합니다. 이를 통해 **디자이너**, **비전공자**, **개발자** 등 다양한 사람들이 소통하는 데 도움을 줄 수 있습니다.

## **핵심 목표**
- 사용자가 제공한 **손그림**을 바탕으로 **요구사항 명세서 초안**과 **다이어그램**을 자동 생성
- **손그림 분석 → 요구사항 추론 → 명세서 및 다이어그램 생성**하는 3단계 기능 구현
- 최종적으로 **텍스트 형식**으로 요구사항을 출력하고, **ASCII 아트**로 시각화된 **프로세스 흐름**을 생성

## **작업 흐름 및 세부 단계**

1. **Step 1: 손그림 업로드 및 UI 컴포넌트 인식**
   - 사용자가 **손그림**을 업로드
   - **OpenCV** 또는 **딥러닝 모델**을 사용하여 손그림에서 **UI 요소**를 추출
   - **JSON 형식**으로 UI 요소 리스트 반환

2. **Step 2: 요구사항 추론**
   - 사용자가 제공한 **목표**(예: "대학생 시간표 앱")와 **UI 요소**를 결합하여 **기능적 요구사항**을 추론
   - **Gemma 3 (4B)** 모델을 사용하여 추론

3. **Step 3: 요구사항 명세서 초안 생성**
   - **마크다운 형식**으로 요구사항을 문서화
   - 예: "로그인 기능", "강의 추가/삭제 기능" 등

4. **Step 4: ASCII 아트 기반 다이어그램 생성**
   - **프로세스 흐름**을 **ASCII 아트**로 표현
   - 예: **순차 다이어그램**, **시퀀스 다이어그램** 등의 텍스트 기반 다이어그램 생성

## **경험담 및 스토리텔링**

이 프로젝트의 아이디어는 **2025년 2학기 현재 수강 중인 소프트웨어 공학 강의**에서의 경험과, **제가 2025년 11월에 참여한 해커톤 대회**에서의 경험에서 비롯되었습니다. 소프트웨어 공학 강의에서 **요구사항 명세서**나 **UML 다이어그램** 등을 다루면서, 실제로 이런 문서들을 효율적으로 작성하는 것이 얼마나 중요한지 깨닫게 되었습니다.

특히, 해커톤 대회에서는 **프론트엔드**를 맡게 되었는데, 이전에 **프론트엔드 경험**이 없어서 **화면 구성**과 **요구사항 정리**가 굉장히 어려웠습니다. **UX 관련 용어**를 몰라서 팀원들과 **화면 구상**을 하는 데에도 어려움이 있었습니다. 그때, 우리는 **손그림**을 그려서 **화면 구상**을 했고, 소통이 잘 되었습니다. 하지만 **"이것을 AI에 어떻게 만들어달라"**고 설명하는 것은 매우 어려웠습니다. 제가 그린 그림을 **전문적인 용어**로 표현하는 것이 너무 힘들었기 때문입니다.

이 경험을 통해, **비전공자**나 **디자이너**들이 **UI 구성**이나 **요구사항 정리**에서 어려움을 겪을 수 있다는 점을 실감했습니다. 그래서 이 문제를 해결할 수 있는 도구가 필요하다고 생각했으며, 바로 이 프로젝트가 탄생하게 되었습니다.

**SketchToSpec** 프로젝트는, **디자이너**나 **비전공자**도 **손그림만 있으면** 요구사항 명세서나 다이어그램을 쉽게 만들 수 있도록 돕는 시스템입니다. 이를 통해, **비전공자**나 **디자이너**도 복잡한 **전문 용어** 없이 소통할 수 있는 환경을 제공할 수 있습니다.

## **결론**
**SketchToSpec**은 **손그림을 기반으로 요구사항 명세서를 자동 생성하는** 혁신적인 시스템을 구축하는 프로젝트입니다. 사용자는 손그림을 업로드하고, 그에 맞춰 자동으로 **요구사항 명세서 초안**과 **ASCII 아트 기반 다이어그램**이 생성됩니다. 이 과정은 **딥러닝 모델**과 **텍스트 추론**을 활용하여 효율적으로 처리됩니다. **Streamlit**을 사용한 **UI**와 **Gemma 3 (4B) 모델**을 활용한 **추론 과정**이 결합되어 **매우 실용적**이고 **창의적인** 시스템을 만들 수 있습니다.
